# 摘要

本报告介绍了校园购物商城系统的设计与实现，这是一个面向大学校园的电子商务平台。系统采用前后端分离架构，前端使用Vue.js，后端采用Spring Boot框架，数据库使用MySQL。系统实现了用户管理、商品管理、购物车、订单处理、AI智能助手等核心功能，满足校园内商品交易的基本需求，并通过AI技术提升用户购物体验。

---

# 第一章 项目背景与意义

## 项目背景

随着电子商务的发展，校园内的商品交易需求日益增长。传统的线下交易方式信息流通不畅，效率低下，缺乏安全保障。因此，开发一个专门的校园购物平台，为师生提供便捷、安全的交易环境具有重要意义。

## 项目意义

1. **提升交易效率**：通过线上平台快速匹配买卖双方，缩短交易时间
2. **保障交易安全**：提供支付担保和交易记录，降低交易风险
3. **促进资源循环**：方便学生之间交易闲置物品，减少资源浪费
4. **实践学习**：将课堂所学知识应用到实际项目中，提高实践能力

# 第二章 需求分析

## 功能需求

1. **用户管理**：注册、登录、个人信息管理、地址管理
2. **商品管理**：商品发布、分类管理、搜索、详情展示
3. **购物车**：添加商品、修改数量、删除商品、结算
4. **订单管理**：创建订单、订单状态管理、订单查询
5. **支付功能**：支持主流支付方式，支付状态处理
6. **评价功能**：商品评价、评价展示
7. **后台管理**：用户管理、商品管理、订单管理、系统设置
8. **AI智能助手**：提供智能聊天功能、购物建议、商品推荐等AI服务

## 非功能需求

1. **性能**：页面加载时间不超过3秒，支持500用户同时在线
2. **安全**：用户认证、数据加密、访问控制、防止常见攻击
3. **可用性**：系统可用性不低于99%
4. **兼容性**：支持主流浏览器和移动设备

# 第三章 系统设计

## 系统架构

采用前后端分离架构：
- **前端**：Vue.js + Element Plus
- **后端**：Spring Boot + MyBatis
- **数据库**：MySQL
- **AI服务**：DashScope API（阿里云大模型服务）
- **部署**：Docker + Nginx

## 模块划分

1. **前端模块**：用户界面、路由管理、状态管理、API交互、AI助手界面
2. **后端模块**：用户服务、商品服务、订单服务、支付服务、管理服务、AI助手服务

## 数据库设计

主要数据表：
- **用户表**：存储用户信息
- **商品表**：存储商品基本信息
- **订单表**：存储订单信息
- **购物车表**：存储购物车数据
- **支付表**：存储支付记录
- **评价表**：存储商品评价

# 第四章 系统实现

## 前端实现

使用Vue.js框架，主要实现：
1. **用户界面**：响应式设计，适配不同设备
2. **组件库**：基于Element Plus构建界面组件
3. **路由**：Vue Router管理页面路由
4. **状态管理**：Vuex管理全局状态
5. **API交互**：Axios处理与后端的数据交互
6. **AI助手界面**：实现智能聊天窗口、购物建议展示等功能

## 后端实现

使用Spring Boot框架，主要实现：
1. **RESTful API**：提供标准化的接口
2. **业务逻辑**：实现核心业务功能
3. **数据访问**：MyBatis操作数据库
4. **安全认证**：JWT认证和授权
5. **异常处理**：统一的异常处理机制
6. **AI助手服务**：集成DashScope API，提供智能聊天、购物建议等功能
7. **文件上传**：支持商品图片等文件上传功能
